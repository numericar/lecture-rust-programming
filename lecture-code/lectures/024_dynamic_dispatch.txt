Dynamic Dispatch
    เป็นการกำหนดพฤติกรรมของ trait แบบ dynamic ในเวลา runtime แทนที่จะกำหนดร่วงหน้าในขั้นตอนการ compile
    โดยใช้ trait object (dyn trait) เพื่อสร้างความยืดหยุนในการจัดการประเภทที่แตกต่างกัน ใน implement trait 
    เดียวกัน

    โดยสิ่งนี้จะใช้ vtable (virtual table) ซึ่งเป็นโครงสร้างที่เก็บข้อมูลรายการของฟังก์ชันที่ประเภทที่ implement trait
    นั้นรองรับ

    เมื่อมีการเรียกใช้งานฟังก์ชัน trait object compiler จะค้นหาและเรียกใช้ฟังก์ชันที่เหมาะสม จาก vtable ในเวลา runtime


    &dyn เป็นการระบุไปยังตำแหน่งที่มีการเก็บฟังก์ชันของ trait นั้นเอาไว้


    -- 
    เป็นการบวนการที่ rust จะตัดสินใจว่าจะเรียกฟังก์ชันอะไรในช่วง runtime แทนที่จะตัดสนใจช่วง compile 
    โดยมักถูกใช้งานกับ trait object 
    โดย dynamic dispatch จะเกิดขึ้นเมื่อ
        1. ฟังก์ชันถูกเรียกใชงานผ่าน dyn trait
        2. โปรแกรมจะใช้ vtable เมื่อระบุและเรียกฟังก์ชัยที่เหมาะสมในช่วง runtime

    vtable  คือโครงสร้างของข้อมูลที่เก็บฟังก์ชันที่เกียวกับ trait นั้น โดยมันทำให้ rust รู็ว่า เมื่อเรียกฟังก์ชันจาก 
            trait object ควรไปเรียกฟังก์ชันตัวไหนในหน่วยความจำ


    แตกต่างจาก static dispatch อย่างไร ?
        1.  static dispatch การที่ rust จะรู้ว่าจะต้องเรียกใช้ฟังก์ชันอะไรจะเกิดขึ้นในช่วง compile time
            แต่ dynamic dispatch เกิดขึ้นตอน runtime 
        2.  ยืดหยุ่นกว่า เพราะสามารถทำงานกับชนิดข้อมูลที่ไม่รู็ล่วงหน้าได้
