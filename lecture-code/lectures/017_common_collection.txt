Common Collection
    ใน rust มี collections ที่ใช้งานกันอย่างแพร่หลาย ซึ่งเป็นโครงสร้างข้อมูลที่ทำให้เรา
    สามารถจัดเก็บข้อมูลได้อย่างมีประสิทธิภาพ

    โดย collection จะอยู่ใน standard library (std::collections) และยัง
    มีความปลอดภัยเพราะใช้ระบบ ownership และ borrowing ช่วยลดข้อผิดพลาด

    โดยจะมีตัวที่นิยมใช้ดังนี้
    
        vectors (Vec<T>)
            เป็น collection ชนิดหนึ่งที่ใช้สำหรับเก็บลำดับของข้อมูลประเภทเดียวกัน โดยมีขนาด
            ที่เปลี่ยนแปรงได้ (dynamic size) ต่างจาก array ที่มีขนาดคงที่ นอกจากนี้
            ยังมีฟังก์ชันและคุณสมบัติที่ช่วยให้เราจัดการข้อมูลได้อย่างสะดวก เช่น การเพิ่ม, ลบ
            และแก้ไขข้อมูลในลำดับได้

            ความสามารถโดดเด่นของ vector 
                1. Dynamic size: เพิ่มหรือลบข้อมูลได้โดยไม่ต้องกำหนดขนาดล่วงหน้า
                2. Efficiet Memory Management: ทำงานอยู่ด้วยกฏ ownership และ borrowing เพื่อจัดการ
                        กับหน่วยความจำ แม้ว่าจะมีการ dynamic ที่ส่วนของ memory 
                3. Generic Type: รองรับการจัดการกับข้อมูลทุกประเภท

    for fruit in &fruits {
        println!("{}", *fruit);
    }
    จากชุดคำสั่งดังกล่าว fruit จะมีชนิดข้อมูลเป็น &&str 
    ทำไมถึงเป็นแบบนั้น ?
    ในขั้นตอนการวนซ้ำ เราได้ขอยืมข้อมูลจาก fruits ด้วยการ borrowing (&fruits) 
    โดยทุก ๆ การวนซ้ำ เราจะได้ fruit มา ซึ่ง fruit นี้จะเป็น reference ที่ชี้ไปหาค่าที่อยู่ใน vector 
    แล้วค่าที่ fruit reference ไปหาก็เป็น string slice ซึ่งเป็น reference เหมือนกัน ทำให้เราได้
    ชนิดตัวแปรมาเป็น &&str

    ** เมื่อทำงานผ่าน collection ค่าที่ได้จากการ loop จะมีการทำ borrowing ให้อัตโนมัติ

    โดยการที่ระบบจะตัดสินใจว่าค่าที่ได้จากการ loop จะเป็น borrowing หรือไม่ จะดูจากการขอซันซ่ำ
    เช่น 
        for item in collection 
        การวนซ้ำแบบนี้่ จะไม่มี & ที่ collection ที่ต้องการวนซ้ำ
        ทำให้จะไม่มีการทำ borrowing ที่ item ให้ โดยเมื่อวนซ้ำแล้วจะมีการเปลี่ยน ownership ของข้อมูล
        จะถูกเปลี่ยน ทำให้ collection เดิมไม่สามารถใช้งานได้อีกต่อไป

        for item in &collection
        การวนซ้ำแบบนี้่ จะมีการทำ borrowing ให้ที่ item จะกลายเป็น &T 

        for item in &mut collection
        การวนซ้ำแบบนี้่ จะมีการทำ borrowing ให้ และสามารถเปลี่ยนแปรงข้อมูลได้ด้วย
