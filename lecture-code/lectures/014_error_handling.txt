Error Handling
    เป็นการจัดการกับสถาณการณ์ที่ไม่คาดคิดหรือการทำงานที่ผิดพลาดของโปรแกรม
    โดยจะมีโครงสร้างข้อมูลอยู่ 2 แบบที่ใช้ในการจัดการข้อผิดพลาด
        1. Result<T, E>     ใช้สำหรับข้อผิดพลาดที่าสามารถคาดการณ์ได้
        2. panic!           ใช้สำหรับข้อผิดพลาด ที่ไม่สามารถกู้คืนได้

    Result<T, E>
        จะเป็น enum ที่มี 2 ค่าหลัก คือ
            1. Ok(T)    สำหรับกรณ๊ที่การทำงานสำเร็จ
            2. Err(E)   สำหรับกรณ๊ที่เกิดข้อผิดพลาด

        รูปแบบของ Result<T, E>

        enum Result<T, E> {
            Ok(T),
            Err(E)
        }

    โดยมีวิธีการจัดการกับ Result ดังนี้
        1. match
        2. .unwrap      ใช้เพื่อดึงค่าจาก Ok ถ้าเกิด Err จะทำให้โปรแกรมเกิด panic (เหมาะสำหรับกรณีที่มั่นใจว่าจะไม่เกิดข้อผิดพลาด)
        3. ? operator   ใช้เพื่อส่งต่อข้อผิดพลาดขึ้นไปให้ function ที่เรียกใช้งานเป็นคนจัดการ
